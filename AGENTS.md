- Use `npm` as the preferred package manager; avoid introducing conflicting lockfiles.

## Security invariants (server-signed / sponsored / relayer transactions)

The server holds a sponsor keypair (`AGENTVOUCH_SPONSOR_*`) and co-signs purchases as fee payer + rent payer in `web/lib/sponsoredPurchase.ts` (`/api/transactions/sponsored/purchase/{prepare,submit}`). Any flow where the server signs a transaction on a user's behalf must hold these invariants — a miss here drains real funds:

- **Never let the user-controlled party equal the sponsor / fee-payer.** If `buyer == sponsor`, the buyer / rent_payer / fee_payer signer slots collapse onto one key, so the server's `partialSign(sponsor)` fully signs the transaction with no user signature — an attacker then drains the sponsor's USDC into an attacker-owned listing. Enforced by `assertBuyerIsNotSponsor` in both the prepare and submit paths (unit-tested in `web/__tests__/lib/sponsoredPurchase.test.ts`); keep that guard whenever you add a server-cosigned flow.
- **The server signs only server-constructed messages, and never re-signs on submit.** `prepare` builds the transaction from on-chain state (not from client-supplied instructions) and is the only place the sponsor signs; `submit` relays and must not add the sponsor signature. `verifySignatures()` then binds the message to what the sponsor signed, so the client cannot tamper after prepare. Never introduce a path that signs a client-supplied transaction (blind signing).
- **Re-validate everything on submit, defense-in-depth.** Compare every account meta (pubkey + isSigner + isWritable) and the instruction data against a freshly-derived expected instruction, enforce the fee cap, and re-check buyer balance — even though the signature already binds the message.
- **Anything the sponsor fronts (rent, fees) must be reimbursed and capped.** The buyer-signed setup-fee transfer reimburses the sponsor in USDC, capped by `AGENTVOUCH_SPONSOR_MAX_FEE_USDC_MICROS`; the amount depends on `AGENTVOUCH_SPONSOR_SOL_USDC_MICRO_PRICE`, a static env. A stale SOL price silently makes the sponsor lose SOL value on every purchase, so keep it current (or move to a live quote) before enabling broadly.
- **Gate these endpoints.** They are unauthenticated and only flag-gated (`AGENTVOUCH_SPONSORED_CHECKOUT_ENABLED` server, `NEXT_PUBLIC_AGENTVOUCH_SPONSORED_CHECKOUT_ENABLED` client). `web/lib/rateLimit.ts` is a best-effort per-instance limiter only — before enabling sponsored checkout broadly in production, add an edge rate limit (Vercel Firewall/WAF) or a shared-store limiter, and consider requiring proof the caller controls `buyerPubkey`.
- **Adding/removing a `purchase_skill` account is a breaking change** and must stay in lockstep across every caller: `programs/agentvouch/src/instructions/purchase_skill.rs`, `web/generated/agentvouch/`, `web/agentvouch.json`, `packages/agentvouch-cli/src/idl/agentvouch.ts` + `src/lib/solana.ts`, both `web/hooks/use{Marketplace,Reputation}Oracle.ts`, `scripts/devnet-usdc-smoke.mjs`, and `tests/helpers/agentvouchUsdc.ts`. The hand-built instruction in `sponsoredPurchase.ts` also hard-codes the 16-account order and `PURCHASE_ACCOUNT_SPACE = 177` (= `Purchase::SPACE`) — update both on any layout change.

## Design Language ("Coral Terminal", web)

The skill detail page (`web/app/skills/[id]/SkillDetailClient.tsx`) is the reference implementation — match it when building or restyling any web surface. Aesthetic: editorial-meets-terminal — serif display + monospace UI chrome, warm coral (lobster) and cool sea accents on a themed surface (near-black dark / warm off-white light). Restrained, trust-first, decluttered. **Light is the default theme** (`next-themes` `defaultTheme="light"` in `app/layout.tsx`); both light and dark must hold. (Web display uses Crimson serif via `font-display`, not the deck's Arial Black; the Inconsolata mono motif is shared with the deck.)

- **Color — never hard-code hex; use the `globals.css` tokens.** Surface `var(--background)` / `var(--foreground)`. Primary accent / CTAs / brand / tags: `--lobster-accent` (coral, `#d95a2b` light, `#f28a61` dark; strong `#fd522e`) plus `--lobster-accent-soft` / `--lobster-accent-border`. Secondary accent / section markers / links: `--sea-accent` (+ `-strong` / `-soft` / `-border`). Status: amber = review, emerald = pass/trusted, red/ember = avoid/fail (Tailwind colors with `dark:` variants). Keep a dominant neutral surface with sharp coral accents — not evenly distributed color.
- **Typography — classes in `globals.css`.** `font-display` (Crimson Text, serif) for page titles and big headings; `font-article` (Crimson Pro, serif) for long-form prose and lead sentences; `font-heading` (Inconsolata, mono) as the UI base — labels, nav, metadata, buttons, code, stats (the load-bearing terminal motif). Rule: serif (**unbolded** — weight ~400/500, never bold) for the _named_ things — page titles, section titles, nav destinations, skill-card titles, and inline links like "See all"; mono for the data layer — eyebrow labels, metadata, stats, code, and small uppercase chrome. Keep Inconsolata UI labels/buttons at normal weight, not semibold. Small eyebrow labels: `text-[11px] font-normal uppercase tracking-[0.14em] text-gray-400` (mono) with a small sea/coral marker icon.
- **Surfaces & components.** Compact cards (skill-preview / action / homepage): `rounded-sm border border-gray-200 bg-white p-4 dark:border-gray-800 dark:bg-gray-900` with a hover lift (`-translate-y-0.5` + coral border + soft coral shadow). Large detail panels (skill-detail sections): `rounded-lg` + `bg-white/70 dark:bg-gray-900/50`. Small controls/inputs/code stay `rounded-sm`; pills `rounded-full`. Tags/chips: coral-soft pill (`rounded-full border border-[var(--lobster-accent-border)] bg-[var(--lobster-accent-soft)] px-2.5 py-0.5 text-[11px] font-medium text-[var(--lobster-accent)]`). Buttons: `lib/buttonStyles.ts` — `navButtonPrimaryInlineClass` (ember/coral solid), `navButtonSecondaryInlineClass` (outline); primary CTAs go full-width inside action cards. Verdict/status pill: bordered color-coded box + big `font-display` word + mono uppercase label. Stat-row: a small grid of big `font-display` numbers over tiny mono uppercase labels. "Decide" layouts: scrollable main column + sticky right rail for the action/trust surface. `components/InfoTip.tsx`: the ⓘ hover/focus tooltip for terse "why" copy. Skill cards (`components/SkillPreviewCard.tsx`, shared by the homepage and `/skills`): author byline cluster (icon + handle) top-left + verdict chip top-right, serif title, serif description, a signals/tags row, then a bottom-anchored stats row (price/action · rep · vouches · downloads, with tooltipped icons).
- **Principles.** Declutter prose — a human shouldn't face a wall of explanatory text, and an agent reads the API/docs/CLI, not the page: cut redundant copy, move "why" into an `InfoTip`, and collapse developer/secondary detail into `<details>` disclosures. Trust-first hierarchy: what it is → who's behind it → can I trust it → what's inside → how to get it. Token-driven and themed: verify both light and dark, never hard-code colors. Restraint over decoration: clean themed backgrounds (no gimmicky glow gradients), purposeful motion only.
